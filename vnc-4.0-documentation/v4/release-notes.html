VNC 4.0 - Release Notes
General Features and Improvements from VNC 3.3.7
  • Support for dynamic resizing of the desktop
  • Support for rendering of cursor by VNC Viewers
  • New F8-Menu
  • New, more flexible and simpler configuration system
  • New, clean code base
  • IdleTimeout: More useful behaviour for idle client detection
  • Improved (host) IP-address based access control system
  • Improved keyboard handling
  • Improved detection of and adaptation to network bandwidth
  • Improved and more configurable brute-force protection
  • Ability to deal with colormaps (palettes) efficiently at the viewer end
  • New and improved Java VNC Viewer
  • Additional colour levels for slow links
  • Better colours on 8-bit palettized viewers
  • Support for "Windows" and Menu keys
  • Viewer rate-limiting of pointer events for slow connections
VNC for Windows Improvements
  • Full support for mult-monitor systems
  • New keyboard mapping algorithms
  • New configuration panel
  • Improved compatibility with unusual colour depths
  • Improved full-screen polling system
  • Improved efficiency with multiple connections active
  • Improved compatibility with applications
  • Improved removal of desktop decorations during sessions
  • Improved handling of colourmapped displays by viewer
  • Improved Viewer rendering efficiency
  • Improved Viewer full-screen mode
  • Support for XP themes
  • VNC Viewer Bell beeps even without a sound card
VNC for Unix Improvements
  • Ability to remote the native X display (":0")
  • Builds more cleanly on many different platforms
  • Improved VNC server configuration
  • Improved support for X selections (cut & paste)
  • Improved inetd integration
  • Improved viewer with better user interface
  • Viewer options can be changed on the fly
  • Improved full-screen mode
Features Not Yet Available in this Release
  • Scaling of remote desktop
